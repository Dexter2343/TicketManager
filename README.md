# Ticket Manager

## Проєкт

Це невеличкий консольний додаток з трьома різними ролями користувачів: **User**, **Admin** та **Owner**.

- **User** – може використовувати команди: `info`, `buy`, `exit`.  
- **Admin** – має ті ж самі команди, що і User, а також доступ до команд: `create`, `clear`, `delete`.  
- **Owner** – особлива роль, яка може реєструвати користувачів з роллю Admin і має доступ до всіх команд консолі. Особливість: лише один користувач може мати роль Owner, зареєструвати ще одного неможливо.

Проєкт містить клас **TerminalRepository**, який реалізує консоль для всіх команд. Для запуску програми використовується клас **RunApplication**.

---

## Команди консолі

- **register** – загальнодоступна команда для створення акаунту новим користувачем.  
- **login** – авторизація користувача незалежно від ролі.  
- **info** – виводить інформацію про користувача: ім'я, прізвище, квиток (звідки і куди), роль.  
- **exit** – закриває програму.  
- **buy** – користувач заповнює дані, отримує квиток, який зберігається в базі даних. Через команду `info` можна переглянути свій квиток.  
- **create** – доступна для ролі Admin; створює авіарейси у базі даних.  
- **clear** – доступна для ролі Admin; видаляє авіарейси за ID з бази даних.  
- **delete** – доступна для ролі Admin; видаляє квитки користувачів за ID.  
- **makeAdmin** – доступна лише для ролі Owner; реєстрація користувачів з роллю Admin.  
- **makeOwner** – може вводити будь-хто, але працює лише якщо в базі ще немає користувача з роллю Owner.
